- [x] 代码随想录 [19]：[删除链表的倒数第N个节点]
	**思路：** 这题考察的是链表操作中一个经典技巧——用双指针在一次遍历中定位倒数第 N 个节点。核心问题在于：单链表只能从头往后走，无法直接"倒着数"。双指针通过制造一个固定间距，巧妙地把"倒数"转化成了"正数"。
	- **核心做法：双指针（快慢指针）+ 虚拟头节点**
	  先创建一个虚拟头节点 `dummyHead`，让它的 `next` 指向原链表头。虚拟头节点的作用是统一处理"删除的是头节点"这种特殊情况，避免额外判断。
	  然后设置两个指针 `fast` 和 `slow`，初始都指向 `dummyHead`。
	  第一步：让 `fast` 先走 n 步。此时 `fast` 和 `slow` 之间就差了 n 个节点。
	  第二步：`fast` 和 `slow` 同步向后移动，直到 `fast->next == NULL`（即 `fast` 指向链表最后一个节点）。此时 `slow` 正好指向待删除节点的前一个位置。
	  第三步：删除 `slow->next`，即 `slow->next = slow->next->next`，并释放被删除节点的内存。
	  最后返回 `dummyHead->next` 即可。
	- **为什么 fast 走 n 步而不是 n+1 步**
	  这是和自己的代码写法相关的一个细节。标准写法里 `fast` 走 n+1 步，配合循环条件 `while (fast != NULL)`，这样结束时 `slow` 正好在待删节点的前一个。
	  而另一种写法（本代码采用）是 `fast` 只走 n 步，配合循环条件 `while (fast->next != NULL)`——当循环结束时 `fast` 指向尾节点，`slow` 同样指向待删节点的前一个。两种写法等价，关键是 **fast 比 slow 多走的路程 = 虚拟头到待删节点前一位置的距离**。
	- **为什么需要虚拟头节点**
	  如果要删除的恰好是原链表的头节点（比如链表长度等于 n），那么待删节点的前一个节点不存在。引入虚拟头节点后，`slow` 初始指向虚拟头，即使删的是原头节点，`slow` 也能正常指向"待删节点的前一个"，逻辑统一，不需要单独处理边界。
	- **时间复杂度**
	  整个过程只遍历了链表一遍：fast 先走 n 步 O(n)，然后 fast 和 slow 一起走到尾 O(L - n)（L 为链表长度），总体 O(L)，即 O(n)。
	  空间复杂度 O(1)，只用了几个指针变量。
	- **边界点**
	  这题的边界主要在两个地方。一是 n 的有效性——题目保证 n 合法（1 ≤ n ≤ 链表长度），所以不需要额外校验。二是内存释放——C++ 中 `new` 出来的节点要手动 `delete` 防止泄漏。代码中先保存 `tmp = slow->next`，修改指针后再 `delete tmp`，顺序不能反。
	- **总结**
	  这题本质上是把"找倒数第 N 个节点"转化为"两个指针保持 N 的间距同步移动"。虚拟头节点 + 双指针是链表题中的常见组合拳，后续很多链表操作（交换、反转、找中点）都会用到类似的思路。

- [x] 代码随想录 [20]：[有效的括号]
	**思路：** 括号匹配是栈的经典应用场景。栈的"后进先出"特性天然适合处理这种"后面遇到的右括号必须匹配最近出现的左括号"的问题。这道题的关键不在于用不用栈，而在于动手写代码之前先把"不匹配"的几种情况分析清楚。
	- **三种不匹配的情况**
	  在开始写代码前，先把可能出错的情况分类，代码才能覆盖全面：
	  1. **左括号多余**：字符串遍历完了，栈里还有没被匹配的左括号（比如 `"(()"`）。
	  2. **括号类型不匹配**：遍历过程中，当前右括号和栈顶的左括号类型对不上（比如 `"(]"`）。
	  3. **右括号多余**：遍历过程中遇到了右括号，但栈已经空了，说明前面没有对应的左括号（比如 `")()"`）。
	  只要代码能覆盖这三种情况，最后栈为空就是合法匹配。
	- **核心做法：栈 + 遇左括号入对应右括号的技巧**
	  常规思路是遇到左括号就把左括号本身压栈，遇到右括号时再判断栈顶左括号是否匹配。但这里有一个很巧妙的优化：**遇到左括号时，不压左括号本身，而是压入它对应的右括号**。
	  这样一来，遇到右括号时只需要比较它和栈顶是否相等即可，不需要额外维护一个"左右括号对应关系"的映射表。代码从"判断对应关系"变成了"判断相等"，简洁很多。
	  具体流程：
	  遍历字符串每个字符 `s[i]`：
	  - 如果是 `'('`，压入 `')'`；如果是 `'{'`，压入 `'}'`；如果是 `'['`，压入 `']'`；
	  - 否则（当前字符是右括号），检查栈是否为空或栈顶是否与当前字符不等。如果栈空（对应情况 3：右括号多余）或不等（对应情况 2：类型不匹配），直接返回 `false`；
	  - 如果栈顶等于当前字符，说明匹配成功，弹出栈顶，继续遍历。
	  遍历结束后，检查栈是否为空：栈非空说明还有左括号没被匹配（对应情况 1），返回 `false`；栈为空则全部匹配，返回 `true`。
	- **奇偶剪枝**
	  一个容易被忽略但很实用的优化：在进入主逻辑之前先判断字符串长度。如果长度是奇数，括号配对一定不可能成立，可以直接返回 `false`。这个 O(1) 的判断能提前过滤掉大量非法输入。
	- **时间复杂度**
	  每个字符入栈/出栈最多一次，遍历一遍 O(n)。空间复杂度最坏 O(n)（全是左括号的情况）。
	- **边界点**
	  空字符串按题目要求视为有效，栈初始为空，遍历也不会进入循环，最终 `st.empty()` 返回 `true`，逻辑自然兼容。
	- **总结**
	  这道题是栈的入门级应用，但其中"压入对应右括号"的技巧值得记住——它体现了"用数据结构本身携带匹配信息"的思想，后续很多对称匹配类问题（如删除相邻重复项、逆波兰表达式）都能复用这个思路。

  **代码位置：**`code/leetcode/2026-05-11-valid-parentheses.cpp`
