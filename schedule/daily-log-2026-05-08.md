- [x] 代码随想录 [5]：[最长回文子串]
	**思路：** 这道题我一共写了三种解法，核心都是围绕“怎么高效判断一个区间是不是回文”来展开。
	- **解法一：动态规划 DP**
	  定义 `dp[i][j]` 表示子串 `s[i...j]` 是否为回文串。
	  状态转移是：当 `s[i] == s[j]` 且 `(j - i <= 1 || dp[i + 1][j - 1])` 时，说明两端字符相同，并且中间部分也是回文，那么当前区间也是回文。
	  因为 `dp[i][j]` 依赖左下方的 `dp[i + 1][j - 1]`，所以要从下往上枚举 `i`，从左往右枚举 `j`。
	  每次发现新的回文串，就更新最长长度和起始位置。
	  时间复杂度 O(n^2)，空间复杂度 O(n^2)。
	- **解法二：中心扩展**
	  回文串具有中心对称性质，所以可以把每个位置当作中心，向两边扩展。
	  对于奇数长度回文，以 `(i, i)` 为中心；对于偶数长度回文，以 `(i, i + 1)` 为中心。
	  扩展过程中只要左右字符相等，就继续向外走，并实时更新最长回文子串的位置和长度。
	  这种做法比 DP 更直观，代码也更短，是这题里很常用的写法。
	  时间复杂度 O(n^2)，空间复杂度 O(1)。
	- **解法三：Manacher 算法**
	  Manacher 的核心目的是在线性时间内求最长回文子串。
	  先在原字符串每个字符之间插入分隔符 `#`，把奇偶回文统一成同一种形式；然后用 `p[i]` 记录以 `i` 为中心的回文半径。
	  同时维护当前回文最右边界 `right` 和对应中心 `center`。如果当前位置 `i` 在 `right` 内，就可以利用对称位置的信息减少重复扩展；只有超出已知范围时才继续真正比较左右字符。
	  最后找到半径最大的中心点，就能还原出原字符串中的最长回文子串。
	  时间复杂度 O(n)，空间复杂度 O(n)。

  **代码位置：**`code/leetcode/2026-05-08-longest-palindromic-substring.cpp`

- [x] 代码随想录 [15]：[三数之和]
	**思路：** 这题本质上是在数组里找三个数，使它们的和等于 0。难点不只是“找到解”，还要“去重”，避免结果里出现重复三元组。我的代码里写了两种做法：哈希表和双指针，前提都是先排序，排序后更容易做剪枝和去重。
	- **解法一：哈希表**
	  先固定第一个数 `nums[i]`，问题就变成在后面的区间里找两个数，使它们的和等于 `-nums[i]`。
	  这里用 `unordered_set` 记录已经遍历过的数字，每次枚举到 `nums[k]` 时，就看集合里是否已经出现过目标值 `target = -(nums[i] + nums[k])`。
	  如果出现过，就说明找到了一个三元组；如果没出现，就把当前数字加入集合，供后面匹配。
	  为了避免重复结果，需要两层去重：
	  第一层是固定数 `i` 去重，如果 `nums[i] == nums[i - 1]` 就直接跳过；
	  第二层是内部遍历 `k` 时跳过连续重复元素，避免相同组合被重复加入结果。
	  时间复杂度 O(n^2)，空间复杂度 O(n)。
	- **解法二：双指针**
	  这也是这题更经典的做法。
	  先排序，然后固定第一个数 `nums[i]`，剩下的区间用两个指针 `left` 和 `right` 分别从两端向中间逼近。
	  如果三数之和大于 0，说明总和偏大，就让 `right--`；如果三数之和小于 0，说明总和偏小，就让 `left++`；如果刚好等于 0，就记录答案。
	  找到一个解后，不能直接只移动一步，而是要先跳过 `left` 和 `right` 旁边的重复元素，再继续收缩区间，这样才能保证结果不重复。
	  同样地，固定数 `i` 也要去重；另外因为数组有序，如果 `nums[i] > 0`，后面所有数都不可能再凑出 0，可以直接结束循环。
	  时间复杂度 O(n^2)，空间复杂度 O(1)。
	- **总结：**
	  哈希表写法更接近“两数之和”的延伸，思路比较自然；双指针更依赖排序后的单调性，代码更简洁，也更适合作为这题的标准解法。

  **代码位置：**`code/leetcode/2026-05-08-3sum.cpp`
