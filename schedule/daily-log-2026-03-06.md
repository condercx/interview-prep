#### 在C语言中，为什么 static 变量只初始化一次？

**【核心原理】**

* **内存分区差异**：`static` 变量存储在**静态存储区**（数据段 .data 或 .bss），其生命周期贯穿整个程序运行期。
* **初始化时机**：静态变量在程序加载时（Load time）或初次进入作用域时由编译器/运行库完成初始化，而非像 `auto` 变量那样在每次函数调用时通过**栈指针偏移**重新创建。
* **持久性**：由于静态区内存在函数退出后不被释放，变量保留了上一次运行的值（记忆功能），后续调用直接跳过初始化语句，访问原内存地址。

**【关键对比】**

| 特性 | static 变量 | auto 自动变量 (局部) |
| --- | --- | --- |
| **存储位置** | 静态存储区 (Data/Bss) | 栈区 (Stack) |
| **生命周期** | 程序开始到程序结束 | 函数调用开始到函数返回 |
| **初始化次数** | **仅一次** | 每次进入声明周期均初始化 |

**【代码精要】**

```c
void counter() {
    static int count = 0; // 只在第一次调用时初始化，位于静态区
    int temp = 0;         // 每次调用都在栈上重新分配并初始化
    count++; temp++;
    printf("count=%d, temp=%d\n", count, temp);
}
// 连续调用三次输出：count=1,2,3; temp=1,1,1

```

#### extern "C" 的作用是什么？

**【核心原理】**

* **名字修饰 (Name Mangling)**：C++ 为了支持函数重载，编译器会将函数名与参数类型组合生成唯一的符号名（如 `_Z5printi`）；而 C 编译器仅保留原名（如 `_print`）。
* **链接匹配**：`extern "C"` 指示 C++ 编译器对该段代码采用 **C 语言的符号命名规则**进行编译和链接，从而消除 C++ 寻找 C 库函数时因“搜寻名不符”导致的链接错误。

**【关键对比】**

| 语言 | 编译后的符号示例 (void func(int)) | 原因 |
| --- | --- | --- |
| **C 语言** | `_func` | 不支持重载，简单映射 |
| **C++ 语言** | `__Z4funci` (取决于编译器) | 支持重载，需区分参数类型 |

**【代码精要】**

```cpp
// 混合编程的标准范式
#ifdef __cplusplus
extern "C" {          // 告诉 C++ 编译器按 C 规则处理
#endif

void cpp_func(int x); // 可被 C 代码识别的函数声明

#ifdef __cplusplus
}
#endif

```

#### const 有什么作用？

**【核心原理】**

* **只读属性**：通过编译器约束，将变量标记为不可修改，违规修改将在编译期报错（具有强类型的安全性）。
* **内存优化**：`const` 定义的常量通常只分配一次内存。相比 `#define` 的物理文本替换，`const` 在某些架构下可被优化入只读数据段（.rodata），且多个引用指向同一地址，减少内存占用。
* **类型安全**：`const` 具有明确的数据类型，而宏只是简单的字符替换，无类型检查。

**【关键对比】**

| 特性 | const 常量 | #define 宏 |
| --- | --- | --- |
| **本质** | 具有类型的变量 (Read-only) | 预处理文本替换 |
| **地址** | **有独立内存地址** | 无地址（仅为字面量） |
| **检查** | 编译阶段进行类型检查 | 预处理阶段简单替换，易出错 |

**【代码精要】**

```c
const int N = 100;     // 必须初始化，之后不可修改
const char* get_str(); // 返回值不可改，必须赋给 const 指针

// 内存优势对比
#define MSG "Hello"    // 每次使用可能产生多个副本
const char* C_MSG = "Hello"; // 全局唯一地址，节省内存

```
