#### C语言宏中 `#` 和 `##` 的用法

##### 1. `#` 字符串化操作符
- **核心功能**：将宏参数转换为字符串字面量
- **基础语法**：`#define STR(x) #x` → `STR(hello)` 展开为 `"hello"`
- **典型应用**：调试宏打印变量名与值
  ```c
  #define DEBUG_PRINT(var) printf(#var " = %d\n", var)
  // 调用 DEBUG_PRINT(count) → printf("count" " = %d\n", count);
  ```
- **关键特性**：相邻字符串字面量会自动连接（`"count" " = %d\n"` → `"count = %d\n"`）

##### 2. `##` 符号连接操作符
- **核心功能**：将两个宏参数拼接成单个标识符
- **基础语法**：`#define CONCAT(a,b) a##b` → `CONCAT(x,y)` 展开为 `xy`
- **空格规则**：`a##b` / `a ## b` / `a## b` 四种写法等价
- **重要限制**：
  - 拼接结果必须是**有效标识符**
  - `##` 会**阻止参数宏展开**（需间接展开时用两层宏）

##### 3. 宏展开注意事项
- `#` 和 `##` 操作时参数**不预先展开**
- 需间接展开时使用辅助宏：
  ```c
  #define EXPAND(x) x
  #define CONCAT_EXPAND(a,b) EXPAND(a)##EXPAND(b)
  ```

#### volatile 关键字含义与三大应用场景

##### 核心语义
- **禁止编译器优化**：每次访问都从内存地址读取，不使用寄存器缓存
- **保证可见性**：确保变量修改对硬件/中断/其他线程立即可见
- **不保证原子性**：多线程场景仍需配合互斥锁使用

##### 场景1：硬件寄存器访问
- **典型代码**：
  ```c
  typedef struct {
      volatile uint32_t SR;  // 状态寄存器
      volatile uint32_t DR;  // 数据寄存器
  } UART_TypeDef;
  ```
- **关键原因**：寄存器值可能被外设硬件异步修改，编译器优化会导致死循环

##### 场景2：中断服务程序共享变量
- **典型代码**：
  ```c
  volatile sig_atomic_t flag = 0;  // 中断标志
  // ISR中: flag = 1;  主循环: while(!flag);
  ```
- **关键原因**：防止编译器将变量缓存到寄存器，导致主程序看不到ISR的修改

##### 场景3：多线程/多任务共享变量
- **典型代码**：
  ```c
  volatile uint8_t *XBYTE = (volatile uint8_t *)0x8000;
  XBYTE[2] = 0x55;  // 每条写操作都生成实际指令
  ```
- **关键原因**：避免编译器合并连续写操作，确保硬件接收到完整指令序列

#### static 关键字的三重作用

##### 1. 函数内静态变量（生命周期延长）
- **核心特性**：
  - 仅初始化一次（程序启动时）
  - 存储在数据段而非栈
  - 函数调用间保持值不变
- **典型应用**：计数器、缓存、单例模式
  ```c
  void counter() {
      static int count = 0;  // 只初始化一次
      count++;
      printf("%d\n", count);
  }
  ```

##### 2. 文件内静态变量（作用域限制）
- **核心特性**：
  - 作用域限制在**当前源文件**
  - 其他文件无法通过 `extern` 访问
  - 避免全局变量命名冲突
- **典型应用**：模块私有状态、文件级缓存

##### 3. 文件内静态函数（链接属性限制）
- **核心特性**：
  - 函数仅在当前文件可见
  - 具有内部链接属性（internal linkage）
  - 编译器可进行更激进优化
- **典型应用**：模块内部辅助函数、隐藏实现细节

##### 关键对比表格
| 特性 | 普通局部变量 | static局部变量 | 全局变量 | static全局变量 |
|------|-------------|---------------|----------|---------------|
| **存储位置** | 栈 | 数据段 | 数据段 | 数据段 |
| **生命周期** | 函数调用期 | 程序运行期 | 程序运行期 | 程序运行期 |
| **作用域** | 函数内 | 函数内 | 全局 | 当前文件 |
| **初始化** | 每次调用 | 仅一次 | 仅一次 | 仅一次 |
| **默认值** | 未定义 | 0 | 0 | 0 |

| 特性 | 普通函数 | static函数 |
|------|---------|-----------|
| **可见性** | 全局可链接 | 仅当前文件 |
| **链接属性** | 外部链接 | 内部链接 |
| **命名冲突** | 可能 | 不会 |
| **优化潜力** | 受限 | 更高 |

        Node* new_top;

            if (old_top == NULL) {
                return false;  // 栈空

            new_top = old_top->next;
        } while (!atomic_compare_exchange_weak(&stack->top, (uintptr_t*)&old_top, (uintptr_t)new_top));
        
        *value = old_top->data;
        free(old_top);
        return true;

    int shared_resource = 0;

    void* reader_thread(void* arg) {
        int id = *(int*)arg;
        
        pthread_rwlock_rdlock(&rwlock);  // 读锁
        printf("读者 %d: 读取值 = %d\n", id, shared_resource);
        sleep(1);
        pthread_rwlock_unlock(&rwlock);
        
        return NULL;
    }

    void* writer_thread(void* arg) {
        int id = *(int*)arg;
        
        pthread_rwlock_wrlock(&rwlock);  // 写锁
        shared_resource++;
        printf("写者 %d: 写入值 = %d\n", id, shared_resource);
        sleep(1);
        pthread_rwlock_unlock(&rwlock);
        
        return NULL;
    }

    void test_rwlock() {
        printf("=== 读写锁示例 ===\n");
        
        pthread_t readers[3], writers[2];
        int reader_ids[3] = {1, 2, 3};
        int writer_ids[2] = {1, 2};
        
        // 创建读者和写者
        for (int i = 0; i < 3; i++) {
            pthread_create(&readers[i], NULL, reader_thread, &reader_ids[i]);
        }
        for (int i = 0; i < 2; i++) {
            pthread_create(&writers[i], NULL, writer_thread, &writer_ids[i]);
        }

            pthread_join(readers[i], NULL);
        }

        }
        
        printf("\n");
    }
