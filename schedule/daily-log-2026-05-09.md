- [x] 代码随想录 [17]：[电话号码的字母组合]
	**思路：** 这题本质上是一个组合问题。输入的每个数字都对应若干个字母，需要从第 1 个数字开始，每次选一个字母，逐层往后拼接，直到把整个 digits 走完，就得到一个完整组合。这个过程很适合用回溯来做。
	- **核心做法：回溯 / 递归**
	  先用一个映射表 `letterMap` 记录数字和字母的对应关系，比如 `2 -> abc`，`3 -> def`。
	  然后定义一个路径字符串 `s`，表示当前已经拼出来的字母组合；再用 `index` 表示当前递归处理到 digits 的第几位。
	  每进入一层递归，就取出当前数字对应的所有字母，依次尝试加入路径；加入后递归处理下一位数字，递归返回后再把当前字母弹出，恢复现场，继续尝试下一个字母。
	- **终止条件**
	  当 `index == digits.size()` 时，说明所有数字都已经处理完了，当前路径 `s` 就是一个完整答案，加入结果数组即可。
	- **为什么这是组合问题**
	  因为每一位数字都要从自己的候选字母里“选一个”，最终结果是多层选择的笛卡尔积，所以天然适合用树形结构去理解。
	  例如 `23`：
	  第 1 层从 `abc` 里选一个；
	  第 2 层从 `def` 里选一个；
	  最后得到 `ad、ae、af、bd、be、bf、cd、ce、cf`。
	- **时间复杂度**
	  设输入长度为 `n`，每个数字最多对应 4 个字母，所以时间复杂度约为 O(4^n * n)。
	  其中 O(4^n) 是组合数量，乘上 O(n) 是构造每个结果字符串的代价。
	  递归栈深度是 O(n)，结果集不计入时的额外空间复杂度也是 O(n)。
	- **边界点**
	  这题要注意空字符串输入。如果 `digits` 为空，通常应该直接返回空结果，不进入回溯。

  **代码位置：**`code/leetcode/2026-05-09-letter-combinations-of-a-phone-number.cpp`

- [x] 代码随想录 [18]：[四数之和]
	**思路：** 这题可以看成是三数之和的升级版。核心还是先排序，然后固定前两个数，把后两个数的问题交给双指针处理。难点主要有三个：去重、剪枝，以及注意整数溢出。
	- **核心做法：排序 + 双重循环 + 双指针**
	  先对数组排序。排序后，一方面方便去重，另一方面也能利用有序性做剪枝。
	  外层先固定第一个数 `nums[i]`，第二层固定第二个数 `nums[j]`，这样剩下的区间里就只需要找两个数，使四个数之和等于 `target`。
	  此时在区间 `[j + 1, n - 1]` 内设置两个指针 `left` 和 `right`：
	  如果当前四数之和小于 `target`，说明总和偏小，`left++`；
	  如果当前四数之和大于 `target`，说明总和偏大，`right--`；
	  如果刚好等于 `target`，就记录当前四元组，然后继续移动指针去找下一个解。
	- **去重处理**
	  这题和三数之和一样，去重非常关键。
	  第一层固定 `i` 时，如果当前值和前一个相同，就跳过；
	  第二层固定 `j` 时，如果当前值和前一个相同，也跳过；
	  找到一个答案后，`left` 和 `right` 也要跳过连续重复元素，避免把同一个四元组重复加入结果。
	- **剪枝思路**
	  因为数组已经排好序，所以可以利用当前固定值和目标值的关系提前结束循环。
	  比如代码里当 `nums[i] > target && nums[i] >= 0` 时，说明后面的数只会更大，不可能再组成目标值，可以直接结束第一层循环。
	  对第二层 `j` 也用了类似的剪枝逻辑：如果当前两个固定数之和已经超过 `target`，并且它们已经非负，那么继续往后只会更大，也可以直接结束第二层循环。
	- **为什么要用 long**
	  四个整数相加时有可能超过 `int` 的范围，所以代码里在求和时转成了 `long`，避免溢出导致比较结果错误。
	- **时间复杂度**
	  排序是 O(n log n)，外层两重循环加内层双指针整体是 O(n^3)。
	  额外空间复杂度是 O(1)，不算结果数组的话只用了几个指针变量。
	- **总结**
	  这题本质上是把三数之和再往前多固定一层，整体套路没有变：排序、固定若干数、剩余部分用双指针、全过程注意去重和剪枝。

  **代码位置：**`code/leetcode/2026-05-09-4sum.cpp`
