#### 什么是哈佛结构和冯诺依曼结构？

答：两者本质区别在于“指令和数据是否共享同一套存储与总线”。

1. 冯诺依曼结构（Von Neumann）
	- 指令与数据统一编址，通常共享同一存储器通路（或同一主存体系）。
	- 取指和访存容易争用带宽，形成经典“冯诺依曼瓶颈”。
	- 优点：结构与编程模型更统一，代码与数据可在同一地址空间灵活组织。

2. 哈佛结构（Harvard）
	- 指令与数据分离编址，使用独立通路访问。
	- 取指和数据访问可并行，吞吐更高，实时性通常更好。
	- 常见于对性能/确定性要求高的嵌入式场景。

3. 典型对比（面试速记）
	- 编址：冯诺依曼统一；哈佛分离。
	- 总线/通路：冯诺依曼共享；哈佛独立。
	- 并行性：冯诺依曼受带宽争用限制；哈佛可并行取指+访存。
	- 场景：冯诺依曼常见于通用计算；哈佛常见于 MCU/DSP 等嵌入式处理器。

4. 与 ARM 的关系（常考）
	- 许多 ARM 内核采用“改进哈佛结构”：
	  - 对程序员呈现统一地址空间（便于软件开发）；
	  - 在微架构层面分离指令缓存（I-Cache）与数据缓存（D-Cache）及访问通路。
	- 所以面试回答时可说：ARM 常见是“统一编址 + I/D 分离通路”的折中设计。

补充：

- 原书里“哈佛结构用于嵌入式、冯诺依曼用于通用计算”是常见工程经验，但不是绝对边界；现代 CPU/MCU 往往是折中与混合实现。

#### 什么是ARM流水线技术？

答：ARM 流水线技术是把一条指令拆成多个阶段并并行重叠执行，以提升吞吐率、缩短平均每条指令执行时间。

1. 基本思想
	- 常见阶段：取指（Fetch）→ 译码（Decode）→ 执行（Execute）。（不同内核可扩展更多级）
	- 多条指令可处于不同阶段并行推进，因此整体吞吐提高。

2. ARM 家族中的典型形态（面试常问）
	- ARM7 常见为 3 级流水。
	- ARM9 常见为 5 级流水。
	- 流水更深通常有助于提高主频与吞吐，但分支失败代价也会更高。

3. 与 PC（R15）的关系
	- 在经典 ARM 状态（A32）下，`PC` 读到的值通常是“当前执行指令地址 + 8”。
	- 在 Thumb 状态（T32）下，`PC` 读值通常是“当前执行指令地址 + 4”。
	- 本质原因：`PC` 反映的是流水线中的取指位置，而不是“刚好正在执行”的那条指令地址。

4. 流水线带来的工程问题
	- 分支跳转会引起流水线刷新（flush），带来性能损失。
	- 数据相关（前后指令依赖）可能引发停顿（stall），需靠转发、重排或等待解决。
	- 中断/异常进入时，硬件会保存返回地址与状态，返回时按异常类型做对应地址修正。

5. 面试可直接复述的一句话
	- ARM 流水线的核心价值是“用阶段并行换吞吐”，代价是需要处理分支和相关性带来的气泡与刷新开销。

补充：

- 原书中“PC 总是当前执行位置 +8”只对经典 ARM 状态普遍成立，不应泛化到所有执行状态与所有内核。
