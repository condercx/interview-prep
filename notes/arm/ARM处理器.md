#### 什么是哈佛结构和冯诺依曼结构？

答：两者本质区别在于“指令和数据是否共享同一套存储与总线”。

1. 冯诺依曼结构（Von Neumann）
	- 指令与数据统一编址，通常共享同一存储器通路（或同一主存体系）。
	- 取指和访存容易争用带宽，形成经典“冯诺依曼瓶颈”。
	- 优点：结构与编程模型更统一，代码与数据可在同一地址空间灵活组织。

2. 哈佛结构（Harvard）
	- 指令与数据分离编址，使用独立通路访问。
	- 取指和数据访问可并行，吞吐更高，实时性通常更好。
	- 常见于对性能/确定性要求高的嵌入式场景。

3. 典型对比（面试速记）
	- 编址：冯诺依曼统一；哈佛分离。
	- 总线/通路：冯诺依曼共享；哈佛独立。
	- 并行性：冯诺依曼受带宽争用限制；哈佛可并行取指+访存。
	- 场景：冯诺依曼常见于通用计算；哈佛常见于 MCU/DSP 等嵌入式处理器。

4. 与 ARM 的关系（常考）
	- 许多 ARM 内核采用“改进哈佛结构”：
	  - 对程序员呈现统一地址空间（便于软件开发）；
	  - 在微架构层面分离指令缓存（I-Cache）与数据缓存（D-Cache）及访问通路。
	- 所以面试回答时可说：ARM 常见是“统一编址 + I/D 分离通路”的折中设计。

补充：

- 原书里“哈佛结构用于嵌入式、冯诺依曼用于通用计算”是常见工程经验，但不是绝对边界；现代 CPU/MCU 往往是折中与混合实现。

#### 什么是ARM流水线技术？

答：ARM 流水线技术是把一条指令拆成多个阶段并并行重叠执行，以提升吞吐率、缩短平均每条指令执行时间。

1. 基本思想
	- 常见阶段：取指（Fetch）→ 译码（Decode）→ 执行（Execute）。（不同内核可扩展更多级）
	- 多条指令可处于不同阶段并行推进，因此整体吞吐提高。

2. ARM 家族中的典型形态（面试常问）
	- ARM7 常见为 3 级流水。
	- ARM9 常见为 5 级流水。
	- 流水更深通常有助于提高主频与吞吐，但分支失败代价也会更高。

3. 与 PC（R15）的关系
	- 在经典 ARM 状态（A32）下，`PC` 读到的值通常是“当前执行指令地址 + 8”。
	- 在 Thumb 状态（T32）下，`PC` 读值通常是“当前执行指令地址 + 4”。
	- 本质原因：`PC` 反映的是流水线中的取指位置，而不是“刚好正在执行”的那条指令地址。

4. 流水线带来的工程问题
	- 分支跳转会引起流水线刷新（flush），带来性能损失。
	- 数据相关（前后指令依赖）可能引发停顿（stall），需靠转发、重排或等待解决。
	- 中断/异常进入时，硬件会保存返回地址与状态，返回时按异常类型做对应地址修正。

5. 面试可直接复述的一句话
	- ARM 流水线的核心价值是“用阶段并行换吞吐”，代价是需要处理分支和相关性带来的气泡与刷新开销。

补充：

- 原书中“PC 总是当前执行位置 +8”只对经典 ARM 状态普遍成立，不应泛化到所有执行状态与所有内核。

#### ARM有几种工作模式？

答：这题要先区分“经典 ARM（AArch32）”和“ARMv8-A（AArch64）”，否则容易答混。

1. 原书要点（经典 ARM）
	- 常见回答是 7 种工作模式：
	- `USR`（用户）、`SYS`（系统）、`SVC`（管理/超级用户）、`IRQ`（普通中断）、`FIQ`（快速中断）、`ABT`（终止）、`UND`（未定义）。
	- 其中除 `USR` 外都属于特权模式；除 `SYS` 外，其余异常相关模式通常归入异常模式。

2. 需要修正/补充的地方
	- “ARM 只有 7 种模式”并不总是正确。
	- 在 ARMv7-A/R 的一些实现中，若开启安全扩展会新增 `MON`（Monitor）；若支持虚拟化扩展会新增 `HYP`（Hypervisor）。
	- 因此 AArch32 语境下常见是“7 种基础模式”，也可能扩展到 8 或 9 种（视架构扩展与实现而定）。

3. 到 ARMv8-A（AArch64）时的回答方式
	- AArch64 不再沿用上述“模式名”体系，而是异常级别 `EL0/EL1/EL2/EL3`。
	- 面试中如果对方问“ARM 有几种模式”，可先反问或主动说明是问 AArch32 还是 AArch64。

4. 面试稳妥答法（推荐）
	- 若题目默认经典 ARM：先答“7 种基础模式（USR/SYS/SVC/IRQ/FIQ/ABT/UND）”。
	- 然后补一句：“带安全/虚拟化扩展时还会有 MON/HYP；ARMv8-A 64 位状态通常按 EL0~EL3 描述。”

补充：

- 一句话版本：经典 ARM 常答 7 种，但现代实现会因扩展与架构代际不同而变化。

#### Arm有多少32位寄存器？

答：这题常见有两个口径，面试时要先说清楚你在用哪个口径。

1. 原书要点（AArch32 经典口径）
	- ARM 处理器共有 37 个 32 位寄存器。
	- 由 31 个通用寄存器 + 6 个状态寄存器组成。

2. 这个“37”怎么来的（关键解释）
	- 可见的通用寄存器名是 `R0-R15`（16 个），但其中一部分在不同模式下有 banked（影子）副本。
	- 经典 ARM（含 `USR/SYS/FIQ/IRQ/SVC/ABT/UND`）下：
	- 通用寄存器物理总数为 31 个。
	- 状态寄存器为 1 个 `CPSR` + 5 个 `SPSR`（FIQ/IRQ/SVC/ABT/UND 各一个），共 6 个。
	- 因此总计 37 个 32 位寄存器（按该代架构与模式集合统计）。

3. 面试里最容易混淆的点
	- “当前模式下程序员可直接访问的寄存器”通常不是 37 个。
	- 大多数时刻你看到的是 16 个通用寄存器 + `CPSR`，异常模式下再有对应的 `SPSR`。
	- 所以“37”是物理寄存器总量口径，不是单一时刻全部可见口径。

4. 代际修正（避免答旧）
	- 该题默认是经典 AArch32 语境（如 ARM9/ARM11 时代）。
	- 到 ARMv8-A 的 AArch64，寄存器体系已变化，通常按 `X0-X30`、`SP`、`PC`、`PSTATE` 与异常级别来描述，不直接套“37”这个答案。

补充：

- 稳妥一句话：经典 ARM（AArch32）常答 37 个 32 位寄存器，但请同时说明“这是物理总量口径，非单时刻可见数量”。

#### Arm2440和6410有什么区别？

答：这题本质是在比较两代三星 ARM SoC 的代际差异，回答时按“内核、主频、多媒体、存储控制器、外设”来讲最清楚。

1. 原书要点（整理后）
	- 2440 主频较低，6410 主频更高。
	- 2440 和 6410 的处理器内核不同。
	- 6410 在视频、多媒体、接口、DMA、图形加速等方面明显强于 2440。
	- 6410 的启动方式和存储支持更灵活。

2. 核心架构差异
	- S3C2440 基于 `ARM920T`，属于较早期的 ARM9 平台。
	- S3C6410 基于 `ARM1176JZF-S`，属于 ARM11 平台，架构代际更新，整体性能与功能更强。
	- 面试里可直接概括为：6410 比 2440 高一代，不只是“主频更高”，而是 CPU 内核、总线和多媒体能力整体升级。

3. 主频与计算能力
	- 2440 常见主频约 400MHz。
	- 6410 常见主频约 533MHz 或 667MHz。
	- 但不要只答主频，真正差距还包括内核代际、缓存/总线设计和专用硬件单元。

4. 多媒体与图形能力
	- 2440 更偏基础控制型/通用嵌入式处理。
	- 6410 集成更强的视频编解码、多媒体处理能力，并支持 2D/3D 图形加速。
	- 因此 6410 更适合做多媒体终端、视频播放、图形界面更复杂的设备。

5. 存储与总线能力
	- 2440 常见为 SDRAM 控制器。
	- 6410 支持 DDR 存储控制，带宽更高。
	- 原书提到 6410 双总线架构，这个点可以保留，但要注意这是实现能力的一部分，不必孤立死记。

6. 外设与启动方式
	- 6410 的外设接口更丰富，DMA 通道更多，启动方式也更灵活。
	- 对 NAND Flash、存储容量扩展和外部多媒体接口的支持也更强。
	- 2440 则更常见于传统教学板、基础裸机/驱动实验平台。

补充：

- 一句话版本：2440 是偏传统 ARM9 的基础型 SoC，6410 是面向更强多媒体与更高性能场景的 ARM11 SoC。
- 原书里“双总线架构”出现了重复条目，已去重；`arm1176ZJF` 更严谨的写法通常是 `ARM1176JZF-S`。

#### ARM指令集分为几类？

答：这题如果按经典 ARM 面试题来答，通常先答 2 类；但更准确的说法要结合具体架构代际。

1. 原书要点（经典口径）
	- 2 类：`ARM` 指令集和 `Thumb` 指令集。
	- `ARM` 指令通常为 32 位定长。
	- `Thumb` 指令早期通常为 16 位，更利于提升代码密度。

2. 为什么原书答案只适合“经典面试语境”
	- 在 ARM7/ARM9/ARM11 这类经典 AArch32 语境下，答“ARM + Thumb 两类”通常没问题。
	- 但后续架构又引入了 `Thumb-2`，它并不是完全独立于 Thumb 的第三大类，更准确说是 Thumb 指令集的扩展，混合 16 位和 32 位编码。

3. 面试时更稳妥的回答
	- 经典回答：`ARM` 和 `Thumb` 两类。
	- 进阶补充：现代 AArch32 常会提到 `Thumb-2`，它让 Thumb 状态下也能拥有更强表达能力和更好的代码密度/性能平衡。
	- 如果问到 ARMv8-A 64 位：通常讨论的是 `A64` 指令集，不再简单套“ARM/Thumb 两类”这个说法。

4. 两类指令集的核心区别
	- `ARM`：32 位定长，功能完整，早期代码可读性和表达能力更强。
	- `Thumb`：早期以 16 位编码为主，代码更紧凑，适合存储资源紧张场景。
	- `Thumb-2`：在保持较好代码密度的同时，引入部分 32 位编码，缩小了与 ARM 状态下功能表达的差距。

补充：

- 一句话版本：经典 ARM 常答 2 类，即 ARM 和 Thumb；如果想答得更完整，再补一句“后续还有 Thumb-2，ARMv8-A 64 位则通常谈 A64”。
