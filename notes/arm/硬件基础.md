#### NAND FLASH 和NOR FLASH异同？

##### 📊 对比表格

| 类别 | NOR Flash | NAND Flash |
|------|-----------|------------|
| **读取方式** | ✅ **随机访问**：像SRAM，可直接指针读取<br>`uint16_t *p = (uint16_t*)0x08000000; val = *p;` | 🔁 **顺序访问**：需通过命令序列<br>`发送读命令→发地址→查状态→读数据寄存器` |
| **读取速度** | ⚡ 快（随机读取~100ns） | 📦 快（页读取~25μs），但**不支持字节级随机读** |
| **写入速度** | 🐌 慢（~10μs/字节），需先擦除 | ⚡ 快（~200μs/页），页编程效率高 |
| **擦除速度** | 🐌 较慢（50ms~500ms/扇区） | ⚡ 快（2ms~5ms/块） |
| **XIP支持** | ✅ **支持**（代码可直接在Flash上执行） | ❌ **不支持**（需拷贝到RAM执行） |
| **可靠性** | ✅ 高，比特错误率<10⁻¹²，通常无需ECC | ⚠️ 较低，比特翻转常见，**必须配备ECC+坏块管理** |
| **接口类型** | 📡 **并行总线**：地址/数据分离，类SRAM接口 | 🔌 **I/O复用接口**：8/16位数据总线，命令/地址/数据复用 |
| **擦写寿命** | 🔁 **1万~10万次**（块级） | 🔁 **SLC:10万次 / MLC:3千次 / TLC:1千次 / QLC:500次** |
| **数据保持** | 🗓️ 10 ~ 20年（-40℃ ~ 85℃） | 🗓️ 1~10年（随P/E次数和温度衰减） |
| **主流容量** | 📦 128KB ~ 2GB | 📦 1GB ~ **32TB+**（3D NAND） |
| **单位成本** | 💰 高（$0.5~5/MB） | 💰 低（$0.01~0.1/MB） |
| **主要用途** | 🔧 代码存储、Bootloader、配置参数、工业控制 | 💾 大容量存储：SSD、手机存储、U盘、视频数据 |

##### 注意：
在典型的嵌入式SoC架构（如S3C2440、STM32等）中，NOR Flash和NAND Flash的"0地址"确实不冲突，因为它们访问机制完全不同。NOR Flash占用BANK地址空间，NAND Flash不占用BANK地址，0地址是内部的。

---

##### 🔧 相同点

```
1️⃣ 非易失性存储：断电数据不丢失
2️⃣ 写入前必须擦除：写操作只能 1→0，擦除才能 0→1
3️⃣ 擦除单元：均以"块(Block)"为最小擦除单位
4️⃣ 存储原理：基于浮栅晶体管(Floating Gate)或电荷俘获(CTF)技术
5️⃣ 接口演进：均发展出SPI等串行接口以降低引脚数（SPI NOR / SPI NAND）
```

---

##### 🎯 选型建议

```
✅ 选 NOR Flash 当：
   • 需要XIP执行代码（如Bootloader、固件）
   • 容量需求 < 256MB
   • 对可靠性/实时性要求极高（汽车、工业）
   • 系统需要快速启动

✅ 选 NAND Flash 当：
   • 需要大容量数据存储（>512MB）
   • 成本敏感，追求$/GB最优
   • 数据顺序读写为主（文件、多媒体）
   • 可接受软件层管理（FTL、ECC、坏块处理）

🔄 混合架构（常见）：
   NOR(启动代码) + NAND(主存储) → 兼顾启动速度与存储容量
```

##### 📈 技术发展趋势（2024+）

| 技术方向 | NOR Flash | NAND Flash |
|----------|-----------|------------|
| **工艺节点** | 45nm~28nm（稳定） | 1xx层→200+层 3D堆叠 |
| **接口升级** | SPI→Octal SPI（8线，~400MB/s） | ONFI 5.0 / NVMe over PCIe 5.0 |
| **新应用** | AIoT边缘代码存储、汽车ADAS | AI训练数据存储、ZNS SSD、CXL内存扩展 |
| **可靠性增强** | 内置ECC、安全启动、双镜像 | LDPC ECC、RAID-like、端到端保护 |
